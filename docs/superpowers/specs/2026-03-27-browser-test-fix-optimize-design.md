# 浏览器端全功能测试/修复/优化设计

日期：2026-03-27  
仓库：toolbox  
范围：浏览器端（Vue 3 + Vite），仅前端

## 目标
- 覆盖“所有功能”测试：全局功能 + 18 个工具的具体用例，确保真实可用。
- 对发现的问题进行根因定位与修复，修复后回归验证。
- 输出清晰可复现的证据（步骤、输入数据、期望/实际、控制台错误）。

## 非目标
- 不涉及后端服务、外部 API 或部署层变更。
- 不进行非必要重构；仅围绕问题根因与可用性改进。

## 范围与覆盖
### 全局功能
- 主题切换（浅/深）与刷新持久化
- 收藏/历史：新增、展示、清空、持久化
- 配置导入/导出：正常文件、空文件、非法 JSON、字段缺失
- localStorage 读写与异常容错（空值/损坏）

### 工具级（18 个工具）
- 每个工具至少覆盖：
  - 正常输入输出
  - 关键边界输入（空、超长、非法格式）
  - 错误提示/校验反馈
  - 清空/复制/格式化/切换等操作

## 测试策略（混合）
1. **自动化（Playwright）**
   - 覆盖：导航、分类展开、主题切换、收藏/历史、配置导入导出、localStorage
   - 覆盖：各工具的主流程（最小可用路径）
2. **手工补测**
   - 覆盖复杂工具/边界用例
   - 覆盖视觉/交互细节（按钮状态、提示文案、滚动与布局）

## 用例粒度与数据
- 每个工具维护“最小可用输入”+“1-2 个边界输入”。
- 统一记录：
  - 输入数据（可复制）
  - 期望行为（输出/提示/格式）
  - 实际行为（含截图或控制台错误）

## 失败处理与修复流程
遵循系统化调试流程：
1. 复现并固定步骤
2. 收集证据（控制台、网络、组件状态）
3. 追踪数据流，定位根因
4. 最小修复
5. 回归验证（目标用例 + 关联用例）

## 优化原则
- 只做可量化、可验证的优化：
  - 明确错误提示
  - 交互一致性（按钮可用状态、清空/复制行为）
  - 性能显著问题（大输入卡顿、明显阻塞）
- 不引入隐藏降级或无证据的“防御性逻辑”。

## 验收标准
- 全局功能可重复通过，刷新后状态一致。
- 18 个工具核心用例与边界用例通过。
- 控制台无未处理错误（如有 warning 需评估不影响使用）。
- 修复后回归通过，不引入新回归。

## 产出
- E2E 执行记录（通过/失败列表）
- 失败用例清单与修复说明
- 回归结果与最终结论
