# 金银铜实时行情工具设计

日期：2026-03-27  
仓库：`toolbox`  
范围：前端工具站新增 1 个浏览器端行情工具，仅前端实现

## 目标
- 新增一个独立工具，用于查询金、银、铜实时价格。
- 展示 `USD/oz`、`USD/g`、`CNY/g` 三组核心价格。
- 补充金、银、铜的趋势图。
- 支持首次进入自动加载、定时自动刷新、手动刷新。
- 在浏览器端对真实可用性进行验证，不做“只显示静态文案”的假功能。

## 非目标
- 不引入后端服务或代理层。
- 不扩展到铝、锌、镍等更多金属。
- 不承诺涨跌幅、K 线、历史走势图；若数据源不稳定提供这些字段，则本期不做。

## 范围
### 标的
- 金：`XAU`
- 银：`XAG`
- 铜：`HG`

### 页面能力
- 初次进入自动请求行情。
- 定时自动刷新。
- 手动刷新按钮。
- 显示最近更新时间。
- 显示加载中、刷新中、失败态。
- 支持复制某个价格值。
- 提供统一时间范围切换：`日 / 周 / 月 / 年`
- 展示 3 张独立趋势图：金 / 银 / 铜

## 数据源方案
### 推荐方案
- 金属实时价格：`https://api.gold-api.com/price/<SYMBOL>`
- 金属历史价格：使用同源历史接口
- 汇率：前端直连一个公开汇率接口，拉取 `USD -> CNY`

### 选择理由
- 金属接口支持浏览器直连，适合当前纯前端架构。
- 金、银、铜都在当前数据源可覆盖范围内。
- `CNY/g` 不能只靠金属源本身得到，必须补一层汇率换算。
- 趋势图与实时价格保持同一来源，避免不同源之间口径打架。

### 数据源限制
- `gold-api` 免费实时价格可直连。
- `gold-api` 历史价格额度较紧，不能把图表请求和自动刷新绑死。
- 因此趋势图只在以下时机请求：
  - 首次进入页面
  - 切换 `日 / 周 / 月 / 年`
  - 点击手动刷新
- 趋势图不参与自动刷新。

### 失败处理
- 金属接口失败：卡片展示错误态，不隐藏失败。
- 汇率接口失败：保留 `USD/oz`、`USD/g`，`CNY/g` 显示不可用并提示原因。
- 部分金属失败：只影响对应卡片，不拖垮整个工具。
- 历史图失败：只影响对应金属图表，不影响实时卡片。

## 数据换算
### 输入
- 金属现价：默认按每金衡盎司（troy ounce）返回的美元价格。
- 汇率：`1 USD = x CNY`

### 输出公式
- `USD/g = USD/oz / 31.1034768`
- `CNY/g = USD/g * usdToCnyRate`

### 显示规范
- `USD/oz`：保留 2 位小数
- `USD/g`：保留 2 位小数
- `CNY/g`：保留 2 位小数

## 组件与模块设计
### 新组件
- `src/components/MetalPriceTool.vue`

职责：
- 驱动页面加载、刷新、自动轮询
- 渲染金/银/铜卡片
- 管理统一时间范围切换
- 渲染金/银/铜三张趋势图
- 呈现错误态、更新时间、刷新状态
- 触发复制与历史记录

### 新数据模块
- `src/utils/metal-price.js`

职责：
- 请求金属价格
- 请求金属历史数据
- 请求汇率
- 做单位换算
- 返回统一结构，避免组件里堆业务逻辑

### 图表子组件
- `src/components/MetalTrendChart.vue`

职责：
- 接收单个金属的历史数据
- 渲染实际价格折线
- 呈现图表空态、错误态、加载态

## UI 设计
### 工具名称
- `金属行情`

### 导航归类
- 放入“验证工具”不合适。
- 放入“数据转换”也别扯淡。
- 推荐新增到“高级工具”或“基础工具”末尾。
- 当前更推荐放入“高级工具”，因为它是外部实时数据型工具。

### 页面结构
- 顶部：
  - 工具标题
  - 数据源说明
  - 自动刷新状态
  - 手动刷新按钮
- 中部：
  - 3 张卡片：金 / 银 / 铜
- 下部：
  - 统一时间范围切换：`日 / 周 / 月 / 年`
  - 3 张独立趋势图：金 / 银 / 铜
- 每张卡片内容：
  - 名称与代码
  - `USD/oz`
  - `USD/g`
  - `CNY/g`
  - 更新时间
  - 复制按钮
  - 错误提示

### 趋势图展示规则
- 每个金属单独一张图，不叠线。
- 展示实际价格折线，不展示百分比走势。
- Y 轴展示真实价格值。
- X 轴按所选范围展示时间刻度。
- 图上只做必要信息，不堆浮夸装饰。

## 刷新策略
- 首次进入立即拉取。
- 自动刷新默认 `60s`。
- 手动刷新随时可触发。
- 正在刷新时禁用按钮并显示状态，避免并发请求打架。
- 组件卸载时清理定时器。

### 实时与趋势分离
- 实时卡片：
  - 首次进入拉取
  - 自动刷新
  - 手动刷新
- 趋势图：
  - 首次进入拉取
  - 切换 `日 / 周 / 月 / 年` 时拉取
  - 手动刷新时拉取
  - 不参与自动刷新

### 图表缓存策略
- 对历史数据按 `symbol + range` 做内存缓存。
- 同一页面会话内重复切换到相同条件时优先读缓存。
- 手动刷新时强制绕过缓存重新拉取。
- 缓存只服务于当前页面会话，不写入 `localStorage`。

## 历史记录策略
- 手动刷新成功时写入历史。
- 自动刷新不写历史，不然历史面板会被刷成垃圾堆。
- 历史记录示例：
  - 类型：`金属行情`
  - 值：`黄金 4430.90 USD/oz`

## 错误处理
- 接口超时、网络失败：显示明确错误文案。
- 汇率不可用：只影响人民币价格，不阻断美元价格展示。
- 返回字段缺失：按失败处理，不默默吞掉。
- 刷新时保留上一轮成功数据，同时展示“更新失败”提示。
- 某个图表请求失败时，保留该图上一轮成功数据；若没有旧数据则展示空态错误卡。

## 测试策略
### 单元级
- 换算函数正确。
- 数据聚合函数在部分接口失败时能输出预期结构。

### 浏览器 E2E
- 首次进入自动加载金银铜三张卡。
- 手动刷新触发新请求并更新页面。
- 自动刷新触发请求。
- `USD/oz -> USD/g -> CNY/g` 展示正确。
- 汇率接口失败时 `CNY/g` 错误态正确。
- 单个金属失败时只影响对应卡片。
- 首次进入自动加载三张趋势图。
- 切换 `日 / 周 / 月 / 年` 会更新三张图。
- 自动刷新不会触发趋势图重复请求。
- 手动刷新会同时更新实时卡片与趋势图。
- 单个金属历史请求失败时只影响该金属图。

## 实现顺序
1. 新增数据模块并定义统一返回结构。
2. 先补失败测试：模块级 + 浏览器级。
3. 新增 `MetalPriceTool` 组件。
4. 注册到 `src/tools/meta.js` 与工具注册表。
5. 补全浏览器端回归。
6. 跑构建和全量 E2E。

## 风险
- 第三方接口可用性不可控。
- 汇率接口如果限制 CORS，需要替换源。
- 铜价数据单位必须确认与金银一致；若源有差异，转换逻辑要按实际字段修正，不能想当然。
- 历史接口额度较紧，若后续用户频繁强刷，可能需要更强缓存或更换历史数据源。
